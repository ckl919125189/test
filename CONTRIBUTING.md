# 贡献指南

欢迎为这个测试仓库贡献代码！以下是贡献指南。

## 开发流程

1. Fork 本仓库
2. 创建功能分支 (`git checkout -b feature/amazing-feature`)
3. 提交更改 (`git commit -m 'Add some amazing feature'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 开启 Pull Request

## 代码规范

- 使用一致的代码风格
- 添加适当的注释
- 编写测试用例
- 更新相关文档

## 提交信息规范

使用以下格式：
```
类型(范围): 简短描述

详细描述（可选）

关联问题: #123
```

类型包括：
- feat: 新功能
- fix: 修复bug
- docs: 文档更新
- style: 代码格式
- refactor: 重构
- test: 测试
- chore: 构建/工具

## 测试要求

- 所有新功能需要包含测试
- 确保现有测试通过
- 测试覆盖率不应降低

## 问题报告

使用 GitHub Issues 报告问题，包括：
- 问题描述
- 重现步骤
- 期望行为
- 实际行为
- 环境信息
